# エージェント使用規定

このプロジェクトでは、タスクに応じて最適なサブエージェントを呼び出しながら作業を進めます。

---

## プロジェクト概要

**目的**: AI活用コンサルティングサービス（建設業特化）の事業設計・マーケティング全般

**対象業務**:
- 事業設計・戦略立案
- ランディングページ制作
- 資料作成（営業資料・セミナー資料）
- コピーライティング
- ターゲット分析・市場調査
- コンサルティング支援

## 使用ルール

### 原則

1. **タスク開始時にエージェント選定を行う**
   - 各タスクの性質を判断し、最適なエージェントを選択
   - 複数エージェントが必要な場合は順序を決めて実行

2. **LP制作はDesign Gym方式を採用**
   - 5役のエージェントを順序立てて使用
   - 戦略 → コピー → デザイン → 実装 → レビュー

3. **並列実行可能なタスクは同時に実行**
   - 独立したリサーチ・分析は並列で効率化

4. **判断が必要な場面ではユーザーに確認**
   - エージェント選定に迷う場合は選択肢を提示

### タスク別エージェント選定ガイド

| タスク | 推奨エージェント | 補助エージェント |
|--------|-----------------|-----------------|
| **事業設計・戦略** | `business-analyst` | `market-researcher`, `competitive-analyst` |
| **ターゲット分析** | `ux-researcher` | `market-researcher` |
| **競合分析** | `competitive-analyst` | `research-analyst` |
| **LP構成設計** | `lp-strategist` | `competitive-analyst` |
| **キャッチコピー作成** | `lp-copywriter` | - |
| **営業資料作成** | `business-analyst` | `sales-engineer` |
| **セミナー資料作成** | `content-marketer` | `technical-writer` |
| **価格設計** | `business-analyst` | `competitive-analyst` |
| **CTA設計** | `lp-strategist` | `lp-copywriter` |
| **FAQ作成** | `customer-success-manager` | `technical-writer` |

## LP制作ワークフロー（Design Gym方式）

```
1. 戦略策定（lp-strategist）
   ↓ セクション構成・導線設計
2. コピー作成（lp-copywriter）
   ↓ copy.md完成
3. トークン定義（lp-design-system）
   ↓ tokens.md完成
4. 実装（lp-implementer）
   ↓ 動作するコード
5. レビュー（lp-qa-reviewer）
   ↓ 改善リスト
6. 反復（2-3回）
   ↓ プロ品質に仕上げる
```
