## セクション1〜4で刺さったポイント

### 1. 3ヶ月AI導入プログラム（15万円×3ヶ月＝総額45万円）

**共感ポイント:**
- 成果が確実に見える・イメージできる設計になっている
- 卒業後の月額5万円チャットサービスは、関係性構築後だからこそ機能する
- 顧問サービスの「ダラダラお金がかかる」リスクを排除できる

**課題認識:**
- 従来の顧問サービスは成果の考え方が抽象的で、何が効果として上がっているか分かりにくい
- 事前に「何を提供するか」を明確にし、3ヶ月というゴールを設定することで買いやすくなる

**発展アイデア:**
- 提供物ベースで価格を変えていく → 単価アップの道筋
- 例: 社員教育を付加 → 3ヶ月で45万円 → 3ヶ月で100万円にスケール可能
- プラン構成は「提供物の積み上げ」で設計する

### 2. MTG回数ではなく「成果」で差別化する

**共感ポイント:**
- 市場はMTG回数の多寡で差別化しようとしている（チャットのみで月10万円のサービスも多い）
- 顧客目線では、MTG回数が増える＝負担が増えるとも言える
- MTG回数以外の「成果の形」を作りたい

**代替価値のアイデアメモ:**
- 電話質問OK（即時対応の安心感）
- 質問に対して動画解説を提供（非同期だが手厚い）
- → 他にもアイデアを膨らませる余地あり

### 3. 1業種特化

**共感ポイント:**
- 1業種特化は競合が手薄で、深い知識による差別化が可能
- 「全業種対応＝浅い」という競合の弱点を突ける

**活用方針:**

**共感ポイント:**

## 壁打ちログ

### Q1: 核心インサイト3〜5の確信度

**回答まとめ:**
- **3（実際にやってくれる）**: 一番できる。ただしキャパ問題あり。業界テンプレート（建設業ならこの改善パターン等）を蓄積していくことで解決できる見通し
- **4（成果が数字で見える）**: 3ができれば4は自然に伝えられる。3と4は連動する
- **5（自走できるようになる）**: これはアップセル/バックエンド商品。上位プランとして「社員教育・内製化」まで持っていく。実現には半年は必要

**導き出された構造:**
```
フロントエンド（MVP）: 3 + 4 = 「実装代行 + 成果の数値化」
バックエンド（上位プラン）: 5 = 「社員教育・内製化支援」（半年プログラム）
```

**未解決の懸念:**
- 1人で受けられる数の限界 → この金額でキャパオーバーにならないか
- 業界テンプレートの蓄積には時間がかかる → 初期はどうするか

### Q2: 1社あたりの工数内訳

**回答まとめ:**

| 業務 | 月あたり時間 | 備考 |
|------|------------|------|
| ミーティング | 2h | - |
| 設定代行 | 10〜20h | **最大のボトルネック。順当にいくか詰まるかで変動大** |
| チャット対応 | 1h | 「ジム理論」で実際はほぼ来ない想定 |
| **合計** | **13〜23h/月** | - |

**キャパ試算（月160h稼働の場合）:**
- 楽観（13h/社）: 最大12社 → 月売上180万円
- 悲観（23h/社）: 最大7社 → 月売上105万円
- 現実的（18h/社）: 約9社 → 月売上135万円

**重要な洞察:**
- **設定代行（10-20h）がサービスの肝であり、最大のボトルネック**
- ここが短縮できるか＝キャパが広がるか＝事業スケールの鍵
- テンプレート化で短縮できれば、同じ価格で利益率が劇的に改善する

**「ジム理論」（チャット・電話し放題の設計意図）:**
- 通い放題のジムのように、使い放題でも実際は消化しない
- 顧客に「いつでも連絡できる」権限を渡すことで安心感を提供
- 結果的にクレームリスクも下がる（「連絡できなかった」不満が消える）
- 固定Zoom MTGは必ず実行されるが、任意のチャット/電話は意外と来ない

### Q3: 設定代行10-20hの中身と期待値調整

**作業内訳（全てやる想定）:**
1. 要件定義・開発計画の策定
2. 業務理解（ここに時間がかかる）
3. AIツールの選定・提案
4. アカウント設定
5. プロンプト作成
6. 内部設計
7. GAS・コーディング・AIワークフロー構築
8. 使い方の解説
9. 業務フローへの組み込み
10. テスト・動作確認

**最大の論点: 期待値調整**
> 「全部やってもらえる」と思われる → がっかりさせる
> 「これしかやらない」と伝える → 大した価値がないと思われる
> **いかに明確に、現実的なラインで、しかも魅力的に伝えるか = サービスの肝**

これがこのサービスの価値そのものであり、最も設計すべきポイント。

### Q4: サービスの型 → B案「段階型」を採用

**選定理由:**
- カスタマイズしている感じが伝わる方が魅力的
- 相手にとって最もインパクトのある業務を選ぶ → 「あなた専用の計画」になる
- 2個目・3個目が残っている状態 → 継続の動機＋期待感になる

**具体的なフロー:**
```
【契約前】要件定義 → 「何を解決するか」の3ヶ月計画を策定・提案
    ↓
【契約判断】3ヶ月計画に納得したら契約・実行
    ↓
【Month 1】最もインパクトのある1業務を完璧に仕上げる
    ↓
【Month 2-3】2つ目・3つ目に展開
    ↓
【保証】計画が実行できなかった場合 → 延長してでも実行する
```

**重要な設計ポイント:**
- 契約前の「3ヶ月計画の提案」がセールスツールになる
- 計画が具体的であるほど顧客の納得度が上がる
- **「達成できなければ延長保証」** → 成果コミットの証明になる
- 2個目・3個目が残る構造 → 卒業後サポート（月5万円）への自然な導線
