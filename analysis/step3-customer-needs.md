# STEP 3: 顧客ニーズ・ジョブ理論の深堀りリサーチ

## 1. 中小企業のAI導入における課題・ペインポイント

### 定量データ（ファクト）

| 課題 | 割合 | 出典 |
|------|------|------|
| 利用用途・シーンがない（何に使えばいいかわからない） | 41.9% | 中小企業330人調査（DXマガジン） |
| 導入・運用コストが不明/高そう | 15.7% | 同上 |
| 社内にAI人材がいない | — | 総務省 令和7年情報通信白書 |
| 効果的な活用方法がわからない | 最多回答 | PwC 生成AI実態調査2025春 |
| セキュリティリスクへの不安 | 2番目 | 同上 |
| ランニングコストがかかる | 3番目 | 同上 |

### AI導入失敗の5大パターン（各種調査の統合）

1. **「目的なき導入」症候群**: 「話題だから」「競合がやっているから」で導入 → 何に使うか決まっていない → 社内定着しない
2. **「誰も使わない」問題**: ツール導入したが現場が使い方を覚えない → 元のやり方に戻る
3. **「効果が見えない」迷走**: ROI測定の設計をせずに導入 → 成果不明 → 経営層の支持喪失
4. **「AI万能」幻想**: AIに過剰な期待 → 期待とのギャップに失望 → 「AIは使えない」結論
5. **「情報格差」の壁**: 大企業にはベンダーから情報が来るが、中小企業は「何ができるか」自体を知らない

### 経営者が感じる不安・障壁

- 「何から始めればいいかわからない」が最大の障壁（推測: 全体の50%以上）
- 「投資した分の元が取れるか不透明」
- 「社員がついてこれるか」
- 「機密情報が漏れないか」
- 「自社に合ったツールがどれかわからない」

## 2. ジョブ理論（Jobs to be Done）分析

### 2-1. 機能的ジョブ（具体的な業務課題を解決したい）

| 優先度 | ジョブ | 具体例 |
|--------|------|--------|
| ★★★ | 定型業務を自動化したい | 議事録作成、メール返信、請求書処理、日報作成 |
| ★★★ | 情報収集・分析を効率化したい | 市場調査、法改正チェック、競合分析、データ集計 |
| ★★☆ | 顧客対応を効率化したい | FAQ対応、問い合わせ分類、チャットボット、営業資料作成 |
| ★★☆ | 人材不足を補いたい | 採用業務効率化、新人教育の標準化、マニュアル作成 |
| ★☆☆ | 新規事業・新サービスを企画したい | AIを活用した新商品開発、業務モデル変革 |

### 2-2. 感情的ジョブ（不安・ストレスを解消したい）

| 優先度 | ジョブ | 心理状態 |
|--------|------|----------|
| ★★★ | AI時代に取り残される恐怖を解消したい | 「周りが進んでいるのにうちだけ遅れている」焦り |
| ★★★ | 「何もわからない」不安を解消したい | AIの全体像が把握できず、判断ができない閉塞感 |
| ★★☆ | 失敗したくない（投資の不安） | 「高いツールを入れて成果が出なかったらどうしよう」 |
| ★★☆ | 信頼できる相談相手が欲しい | ベンダーは売りつけてくる、社内には詳しい人がいない |
| ★☆☆ | 社員からの突き上げに応えたい | 「うちの会社もAI使いましょう」と言われるが判断できない |

### 2-3. 社会的ジョブ（対外的な評価・イメージ）

| 優先度 | ジョブ | 対外的期待 |
|--------|------|-----------|
| ★★☆ | DX先進企業として見られたい | 取引先・顧客からの信頼獲得、採用競争力の向上 |
| ★★☆ | 業界内で「先を行っている」存在になりたい | 同業者からの尊敬、業界団体での発言力 |
| ★☆☆ | 補助金・認定を活用して対外アピールしたい | DX認定、IT導入補助金採択をPR材料にしたい |

### ジョブの優先順位（推測）
```
最も強いジョブ:
「AIで何ができるか教えてほしい」+「失敗せずに第一歩を踏み出したい」

→ つまり顧客が本当に求めているのは
  「ツール」ではなく「安心して任せられるガイド」
```

## 3. 購買意思決定プロセスの分析

### 3-1. 起案者と決裁者（企業規模別）

| 企業規模 | 起案者 | 決裁者 | 特徴 |
|---------|--------|--------|------|
| 10人以下 | 社長本人 | 社長本人 | ワンマン決裁。社長の「やりたい」で即決 |
| 10〜50人 | 社長 or 総務・経理 | 社長 | 社長の理解が全て。現場の声は限定的 |
| 50〜300人 | 情シス or 経営企画 | 役員会 | 稟議が必要。ROI説明資料が求められる |
| 300人以上 | DX推進部 or 情シス | 経営会議 | 複数部署の調整が必要。導入まで時間がかかる |

### 3-2. 比較対象（何と比較検討されるか）

```
AI顧問サービスの競合は「他のAI顧問」だけではない

1. 何もしない（最大の競合）    ← 60%以上がここ（推測）
2. 自社で独学（YouTube/Udemy） ← コスト意識の高い経営者
3. SaaSツール直接導入          ← 「ツールだけあれば十分」層
4. フリーランスAIコンサル       ← 知人紹介、安価
5. 他のAI顧問サービス          ← 直接比較
6. 大手コンサル（Accenture等）  ← 予算のある中堅以上
7. 社内AI人材の採用            ← 長期的視点の企業
```

### 3-3. 決裁のボトルネック

| ボトルネック | 深刻度 | 対策の方向性 |
|-------------|--------|------------|
| 「成果が出るかわからない」 | ★★★ | 成果保証、無料トライアル、事例提示 |
| 「月額○万円は高い」 | ★★★ | 補助金活用、ROI試算、時間換算での価値提示 |
| 「忙しくて検討する暇がない」 | ★★☆ | 最小限の負荷で始められる設計、初回無料診断 |
| 「社内に説明できない」 | ★★☆ | 決裁用資料テンプレート、上司説得パッケージ |
| 「セキュリティが心配」 | ★☆☆ | セキュリティガイドライン提供、情報管理体制の明示 |

### 3-4. 行動トリガー（何がきっかけで動くか）

```
【外部トリガー】
- 同業他社がAI導入して成果を出したニュースを見た
- 取引先から「AI対応してほしい」と言われた
- 業界団体のセミナーでAI事例を聞いた
- 補助金の募集が始まった
- 法改正（電帳法、インボイス等）への対応が迫られた

【内部トリガー】
- 人が辞めて業務が回らなくなった
- 繰り返しの単純作業にうんざりした
- 社員から「AIを使いたい」と提案された
- 経営数値が悪化し、コスト削減が急務になった
```

## 4. 企業規模別のニーズ差分

### 10人以下（零細企業）
- **最大のニーズ**: 社長自身の業務効率化（メール、資料作成、情報収集）
- **予算感**: 月額1〜5万円が限界
- **意思決定**: 社長の直感。「面白そう」「役に立ちそう」で決まる
- **支援に求めるもの**: 手取り足取りの操作指導、すぐに使える具体的テクニック
- **障壁**: 「社長が忙しすぎて学ぶ時間がない」

### 10〜50人（小企業）
- **最大のニーズ**: 特定業務の効率化（経理、営業、カスタマーサポート）
- **予算感**: 月額5〜15万円
- **意思決定**: 社長判断だが、現場の意見も聞く
- **支援に求めるもの**: 業務分析→ツール選定→導入→定着までの伴走
- **障壁**: 「担当者を決められない」「全員が忙しい」

### 50〜300人（中企業）
- **最大のニーズ**: 組織全体のAI活用戦略、複数部署への展開
- **予算感**: 月額15〜30万円
- **意思決定**: 稟議制。経営企画or情シスが起案、役員会で承認
- **支援に求めるもの**: 戦略立案、ROI試算、社内研修、ガイドライン策定
- **障壁**: 「部署間の温度差」「情シスの人員不足」「既存システムとの連携」

### 300人以上（中堅企業）
- **最大のニーズ**: 全社DX戦略の中でのAI活用位置づけ、ガバナンス構築
- **予算感**: 月額30〜50万円以上
- **意思決定**: DX推進部or経営企画が主導、経営会議で決裁
- **支援に求めるもの**: 戦略コンサルティング、PoC支援、セキュリティ・ガバナンス
- **障壁**: 「部門間調整」「既存ベンダーとの関係」「コンプライアンス要件」

## 5. 既存AI顧問サービスへの不満・潜在ニーズ

### 5-1. 推測される主な不満（Web検索・市場分析から推測）

| 不満カテゴリ | 具体的な声（推測） | 根拠 |
|-------------|------------------|------|
| 成果が曖昧 | 「MTGは毎月やるが、具体的に何が変わったかわからない」 | AIプロジェクトの80-90%がROI未達成（PwC調査） |
| MTGだけで終わる | 「話は聞いてくれるが、実際にやってくれるわけではない」 | 全社が「相談」ベースで実装支援は別料金 |
| 業界知識がない | 「AIの話はできるが、うちの業界のことがわかっていない」 | 15社中14社が「全業種対応」 |
| 社内に定着しない | 「顧問がいる間は進むが、MTGの間に元に戻る」 | 定着支援の仕組みが薄い |
| 担当者の質にバラつき | 「最初は詳しい人が来たが、途中から若手に代わった」 | 顧問モデルの構造的課題 |

### 5-2. 推測される解約理由

1. **「成果が見えない」** — 3-6ヶ月経っても具体的な改善が数値化されない
2. **「自分たちでできるようになった」** — ChatGPT等のツールが使えるようになり卒業
3. **「忙しくてMTGに出られない」** — MTGベースの支援モデルが顧客の負荷に
4. **「コストに見合わない」** — 月額10万円×6ヶ月=60万円の投資対効果が不明
5. **「次のアクションがわからない」** — 相談はできるが実装は自社で、そこが進まない

### 5-3. 潜在ニーズ（「こんなサービスがあれば使いたい」）

| ニーズ | 具体的なイメージ | 優先度 |
|--------|----------------|--------|
| **成果保証型** | 「○○業務の工数を30%削減します。達成しなければ返金」 | ★★★ |
| **ハンズオン実装型** | 相談だけでなく、実際にAIツールの設定・カスタマイズまでやってくれる | ★★★ |
| **業界特化型** | 自分の業界の事例・テンプレートがすでに用意されている | ★★★ |
| **非同期サポート** | MTGの時間を取らなくても、動画やチャットで進められる | ★★☆ |
| **卒業プログラム** | 3-6ヶ月で社内で自走できる状態にしてくれる（永久依存しない） | ★★☆ |
| **コミュニティ型** | 同業の導入企業同士で情報交換できる場がある | ★★☆ |
| **AIダッシュボード** | 導入効果がリアルタイムで可視化される | ★☆☆ |

## 6. 統合インサイト

### 顧客が本当に求めているもの（推測）

```
求めているのは「AI顧問」ではない。
求めているのは「AIで確実に成果を出してくれる人」。

具体的には:
1. 自分の業界・業務を理解した上で
2. 何をすべきか明確に教えてくれて
3. 実際にやってくれて（or やり方を手取り足取り教えてくれて）
4. 成果が数字で見えて
5. 最終的には自分たちでできるようになる

→ 既存サービスは1-2しか提供できていない。3-5が圧倒的に不足。
```
