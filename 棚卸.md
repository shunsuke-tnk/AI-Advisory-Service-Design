> **実施日**: 2026-02-18

## 1. 業界の課題棚卸し（建設業・リフォームのペイン）

### 1-1. 書類・事務作業の非効率（最大のペイン群）
- **見積もり作成が深夜に及ぶ**: 現場から帰社後に初めて着手。夜中まで続く
- **紙→PC転記の二重作業**: 現場で紙に記録 → 事務所でPCに打ち直し
- **社内申請が全て紙**: 小さな申請も紙ベース。デジタル化されていない
- **FAXのやりとり → 紙フォルダ管理**: 受信FAXを物理フォルダに整理
- **資料作成の負担が大きい**: 提案書・報告書の作成に多大な時間
- **現場では書類業務が一切できない**: 通常勤務＝現場、書類は残業時間に

### 1-2. コミュニケーション・調整の非効率
- **下請業者との電話やりとりが多すぎる**: 数ある下請との個別連絡
- **複数人の日程調整**: お客さん × 業者 × 自分の三者調整
- **工事中止連絡の漏れ**: 連絡がめんどくさい・忘れるなどのヒューマンエラー

### 1-3. 営業・提案の課題

#### 【深掘り】見積もりワークフローの全体像（1件あたり2〜3時間）

```
① 現場調査 → お客さんと日程調整が必要
② 工事項目の設計 → 何の工事が必要か、どの職人が担当するか調べる
   └ 初心者は「何を聞けばいいか」すらわからない（最大の学習コスト）
③ 各職人への連絡・見積依頼 → 現地に出向く必要あり、日程調整が再発生
④ 職人からのFAX見積もりを待つ → 待ち時間が発生
⑤ FAX見積もりの転記 → 手入力で写す（地味だが大きなストレス）
⑥ 積算・入力転記 → 最も時間がかかる工程
⑦ 原価表との照合 → 価格を調べるのに時間がかかる
⑧ 価格調整 → 利益率計算、値引き判断
```

**構造的問題**: テンプレ化すると「しょぼい見積もり」になる。時短と品質の両立が難しい。

**解決した方法**: AIにExcel関数を聞きまくり、ドミノ倒しのように関数同士をつなげた。最初の入力だけで全てが連鎖計算されるシートを構築 → 時短と複雑な見積もりを両立。

- **カタログ地獄**: 商品ごとに大量のカタログ、持ち歩きの負担
- **色・仕上がりの不満が多い**: 事前にイメージを伝えきれない
- **提案品質 vs 手間のトレードオフ**: 顧客体験を上げるほど手間が増える構造
- **営業教育に時間がかかる**: 新人育成の工数が大きい

### 1-4. ファイル・情報管理の課題
- **ローカルフォルダがパンク**: 容量不足、動作が遅い
- **ファイルの所在不明**: どこに何があるかわからない
- **紙資料が物理的に場所を圧迫**: オフィススペースの問題

### 1-5. 現場管理の課題
- **遠隔での現場管理ができない**: 現場に行かないと状況がわからない

### 1-6. 人材・組織の課題
- **営業の教育に時間がかかる**: ノウハウの属人化、標準化の難しさ

## 2. 自分の強み棚卸し（Unfair Advantage）

### 2-1. 建設業での実体験・実績
- リフォーム営業5年間、年商2億円超のトップセールス
- 見積もり2〜3時間 → Excel関数の連鎖設計で大幅時短を実現（時短と品質の両立）
- 見積もりワークフローの全工程を体験的に理解している

### 2-2. AI活用の実績

#### 業務に「AIを組み込んだ」もの
- **見積もりExcelシート**: AIに関数を聞き、ドミノ倒し式の自動計算シートを構築
- **クレーム返信用Bot**: 定型的なクレーム対応を自動化
- **営業アドバイスAI**: 営業判断のサポート
- **細かな提出物のフォーム管理**: 業務プロセスの仕組み化

#### AIを「ツールとして使った」もの
- **ブログSEO（ChatGPT 4o）**: 3記事連続でSEO 1位を獲得 ★具体的成果
  - アナリティクス分析 → キーワード分析 → タイトル生成 → 記事作成 → サムネイル作成
  - **一連の動作を全てフォーマット化**（再現可能なワークフローに落とし込んだ）
- **議事録・音声文字起こしと分析**: 顧客やりとりの記録
- **経営戦略の壁打ち**: 意思決定のサポート
- **広告クリエイティブの作成**: マーケティング素材

#### 【重要な気づき】
> **AIを使って「改善策を考えた」経験は多いが、AI自体を業務に「組み込んだ」ものは少ない。**
>
> → つまり自分の強みは「AIエンジニアリング」ではなく、
> **「業務課題を理解し、AIを使って解決の仕組みを設計する力」**にある。

### 2-3. 仕組み化の実績（AIの有無を問わず）

#### パターン: 「分解 → ボトルネック特定 → 最小入力で最大出力の仕組みを作る」

| 対象 | やったこと | 仕組み化のレベル |
|------|-----------|---------------|
| **広告** | 広告投函→最終粗利を一列の数式に。変数/定数に分解し、インパクト大×即効性の変数に集中改善 | 数値で回るPDCA |
| **失注分析** | ToC営業の失注要因を7つに分類。**埋める順番**が重要: ①時期→②自分→③キーマン→④金額→⑤プラン→⑥競合→⑦自社（+先回りクロージング）。1つでも欠けたら失注。テストクロージングで各要因を潰す。詳細は3-9参照 | 体系化された改善フレーム |
| **営業ステップ** | セミナー→調査→テストクロージング→クロージングの4段階に分解。どのステップがボトルネックか特定→改善 | ファネル最適化 |
| **提案資料** | 選ぶだけで完成する資料に設計 | 入力最小化 |
| **見積書** | 数字を入れるだけで完璧な見積書が出る | 入力最小化 |
| **ブログ** | テーマを選ぶだけで記事が書ける（SEO 1位×3記事） | ワークフロー自動化 |

**共通点**: 全てにおいてAIへの壁打ちがベースにあった。

#### 【強みの本質の定義】

> **自分の強みは「AI技術」でも「営業力」でもなく、**
> **「複雑な業務を分解し、ボトルネックを見つけ、最小入力で回る仕組みに再設計する力」。**
> **AIはその仕組み化を加速するツールとして使っている。**
>
> これを建設業の現場知識（5年・年商2億超）と掛け合わせているのが、代替不可能なポジション。

### 2-4. 現在の技術力（独立後に習得）

> **過去**: AIを壁打ち相手にして仕組みを作った（AI × 業務改善）
> **現在**: AI自体を業務に組み込む技術を習得（AI実装力）

#### 習得済みの技術スタック
- **API連携**: 各種AI APIの直接利用
- **Dify**: ノーコード/ローコードAIワークフロー構築
- **Claude Code**: AIエージェント開発
- **Workspace Studio**: （詳細確認中）

#### 目指していること
- **キラーコンテンツとなるデモツールを複数作成**: 顧客に「これが欲しい」と思わせる実物
- **業界特化の改善パターンを構築**: 開発効率化のためのテンプレート群

### 2-5. ネットワーク・つながり

#### 紹介元
- **元タマホーム役員の建設業コンサルタント**: そのクライアント企業への紹介ルート

#### 最初のターゲット顧客像

- **その他**: 解体業者、工務店、リフォーム店。年商数億円規模
- **関係性**: 完全新規（コンサルタント経由の紹介）

#### IT環境（コンサルタントからのヒアリング）

- AndPadなどの業界ツールを導入している企業もあるが、**使いこなせていない**

> → 「AndPadを入れたけど使いこなせていない」という状況は、AI顧問の価値を示す好機

## 3. デモツール構想一覧（ペイン × 提供価値のマッチング）

> **導入ボトルネック**: どんなに良いツールでも、アナログに慣れている人の「導入抵抗」がネックになる。
> → ツール設計は「今の業務を変えない」「入力を極限まで減らす」が鍵。

### 3-1. 見積もり関連 ★最大ペイン
- **AI見積書生成**: 競合に勝てる「顧客が驚くレベルの見積書」を一瞬で作る
- 高粗利率を獲得するためのノウハウも組み込み可能（AI以外の付加価値）

### 3-2. 二重転記の自動化（広範囲に刺さる）
- **各種報告書**: フォーム・LINE・音声で入力 → APIで文章化 → 所定書類に自動転記（カスタマイズ対応）
- **小口精算**: 写真を送るだけで完了
- **運転記録・日報**: 最小入力で記録完了
- **顧客管理シート**: Excel/Notion/スプシを変えずにAIで高機能化（既存ツールを壊さない）

### 3-3. 日程調整の自動化
- **複数人日程調整**: 調整さん方式（カレンダーアカウントがバラバラでも対応）
- 選択 → 確定 → 個々にメール通知が自動で飛ぶ

### 3-4. カタログのデジタル化
- **独自Webサイト形式で全カタログを一括管理**: ページごとに整理、軽量、DL可能
- パッと見れることが最優先（UX重視）

### 3-5. ビジュアルシミュレーション（顧客体験の革新）
- **内装シミュレーションアプリ**: 色・素材を変えてプレビュー
- **外装シミュレーションアプリ**: おうちの着せ替え
- **3D CAD**: 実現できたら最強（技術難易度高）

### 3-6. 提案資料の自動生成
- Skills等を活用し、少ない入力で完璧な見た目の資料が完成するツール

### 3-7. ファイル移行・管理
- **ローカル → Google Drive自動書き出しツール**: 移行・バックアップの楽さ

### 3-8. 遠隔現場管理 ★高ポテンシャル
- **日報フォーム**: 音声・トグル・チェックボックス・短テキストのみ（極限まで簡単）
- **工程表ガントチャートのスピード作成** ★実現したい
- **リアルタイム工程表更新**: 日報や監督の簡単操作で工程表が自動更新 ★めちゃくちゃいい
  - 顧客にも共有可能 → 「今何やっているか」の見える化
  - 安心感 + 「頑張ってくれてる」という信頼構築

### 3-9. 営業教育ツール ★独自性が高い
- **営業通信簿**: 音声入力 → AIが自動分析 → 各項目ごとに個別評価・フィードバック
  - 営業ステップ4段階のどこがボトルネックかを特定
  - 1枚のわかりやすいレポートとして出力

#### 【核心】ToC営業の7つの失注要因 — 評価ロジックの全容

> **原則**: 7つの要因のうち、**1つでも満たされなければ失注する**。
> トップセールスは商談前に各要因を**テストクロージング**で潰してから本商談に臨む。
> → AIは「各要因がテストクロージングできていたか」を判定する。

**埋める順番（＝重要度順）**:

**① 時期**（最重要）
- 「今すぐやりたい」になっていないと、絶対に買わない。それがリフォーム
- 「今すぐ」を実感させるのが**資料（視覚情報）**の役割
- → テストクロージング: 顧客が「今すぐ」の状態にあるかを確認

**② 自分**
- 「この人から買いたい」と思わせること
- 2つの軸がある:
  - **人柄・姿勢による信頼**: 否定せず肯定する、褒める、相手に話させる、質問に徹底的に答える
  - **専門性**: 業界知識・提案力で「この人はプロだ」と思わせる
- → テストクロージング: 信頼関係が構築されているかを確認

**③ キーマン**
- ToC営業の場合、決裁者＝**家族（特に夫婦）**
- **アポの段階で夫婦を揃える**ことが必須
- → テストクロージング: アポ取り時に決裁者全員の同席を確保できたか

**④ 金額**
- 「今すぐやろう」と本気になると、人は「いくらかかるんだろう」と思い始める
- つまり、**値段について質問がなければ本気じゃない可能性が高い**
- 手順: 値段の質問を**引き出す** → **値段教育**を入れる → 金額に対する**テストクロージング**をかける

**⑤ プラン**
- 見積書にロジックがある
- **誰もが完璧な見積書を作れるツール**を提供する → 新人でも契約を上げる仕組みに直結
- → テストクロージング: 提案内容に納得しているかを確認

**⑥ 競合**
- 他に**相見積もり**があるのかどうかを確認する
- セオリー: **最後に出す**（他社の見積もりを見てから提案できるポジションを取る）
- → テストクロージング: 競合状況を把握し、自社の優位性を示せているか

**⑦ 自社**
- そもそも、**自社に圧倒的な不信感がないか**を確認
- 会社の評判・実績・口コミなどが障壁になっていないか
- → テストクロージング: 会社への信頼があるかを確認

**+1: 先回りクロージング**（7つを埋めた後の最終確認）
> 「気に入っていただけたら次回契約でよろしいですね。」（はい、を待つ）
> 「もちろん、気に入らなかった場合は断っていただいて結構です。」
> 「ただ、気に入ったときはぜひ弊社とご契約をお願いします。」
> 「よろしいですか？」（はい）← ここまで待つ

**AI実装の設計思想**:
- 7つの要因ごとに**専門AIが個別に判定**（1AI＝1要因で並列稼働）
- 長文を読んでダラダラと評価するのでは精度が上がらない
- 「テストクロージングできたか否か」というシンプルな判定を7つに分けることで、**尖った精度の評価**が実現する
- 出力: 弱み分析 / 即決オファーパターン提案 / 反論パターン / 御社独自の営業ロジック

### 3-10. 各種チャットBot

- **営業向けチャットBot**: 商品知識・提案パターンのサポート

### 3-12. 外注業者プラットフォーム

### 4-2. セールスプロセス

```
① 相談・ヒアリング（無料）
  └ コンサルタント経由の紹介 → 初回面談
② デモ（無料）
  └ キラーコンテンツのデモツールを見せる
③ 要件定義 ★有料/無料を検討中
  └ 課題ヒアリング → 3ヶ月改善計画の策定
  └ 「何をどう改善するか」を具体的に計画化
④ 契約
  └ 計画の価値に納得した顧客のみが契約
⑤ 実行（3ヶ月）
  └ 計画に基づくツール導入・業務改善の伴走
  └ 成果コミット: 3ヶ月後に計画目標の達成
```

### 4-3. 提供範囲（0→1フェーズの方針）

**現在の考え**:
- **最初は属人的に深く入る**: ツール導入・定着まで身を入れて伴走
- **徐々に仕組み化**: 属人性を抜きながらスケーラブルな形に移行
- **キャパの課題**: 1人でどこまでできるかの制約あり

**顧客の本当の悩み**: 「導入して終わり」ではなく「定着まで支援してほしい」
→ ここに応えることが差別化ポイント

### 4-4. 継続モデル（3ヶ月サイクル）

```
【3ヶ月の伴走期間】
  │
  ├── パターン①: 卒業 → 月額5万円のチャットサポートにダウングレード
  │    └ 軽い相談・メンテナンス対応
  │
  └── パターン②: 継続 → 別の課題で新たな3ヶ月計画を策定
       └ 再度伴走型で次のテーマに取り組む
```

### 4-5. デモツール優先順位（初回商談用）

#### 最優先（必ず持っていく3つ）

| 優先度 | ツール | 刺さるポイント | 訴求軸 |
|--------|--------|--------------|--------|
| ★1 | **AI見積もり作成** | 最大ペイン直撃。「2〜3時間 → 数分」のインパクト | 売上・利益 |
| ★2 | **工程表自動更新ツール** | 現場管理の見える化。顧客共有で信頼構築にも | 管理・効率 |
| ★3 | **営業通信簿AI** | 音声入力 → HTML形式のフィードバック。独自性が高い | 人材育成 |

#### 【戦略的分析】この3つが最強の組み合わせである理由

```
★1 AI見積もり = 「売上・利益を上げる」ツール（攻め）
  → 経営者が一番食いつく。ROIが即座に見える
  → あなた自身の原体験（5年間・年商2億）が裏付け

★2 工程表自動更新 = 「管理コストを下げる」ツール（守り）
  → 拡大中の企業（滋賀のリフォーム店）に特に刺さる
  → 顧客への共有機能 = 営業ツールにもなる

★3 営業通信簿AI = 「人を育てる」ツール（人材）
  → 建設業最大の構造問題「人材」に直接アプローチ
```

## 5. リサーチファクト（市場データ・業界動向）

> 詳細: [リサーチ_建設業界課題.md](リサーチ_建設業界課題.md)

### 人手不足（最重要ファクト）
- 有効求人倍率: **4.81倍**（躯体工7.65倍）— 人を採りたくても採れない
- 就業者数: 685万人(1997) → **483万人(2023)** — 30%減
- 55歳以上: **36.6%** / 29歳以下: **11.6%** — 高齢化
- 新卒3年離職率: 大卒**30%**、高卒**40〜50%** — 若手が定着しない
- 2025年に**約90万人不足**と予測
- 大手の**7割が2026年度に大型工事を新規受注できない**

### 2024年問題
- 時間外労働の上限規制が2024年4月〜適用
- 残業を減らせ → 人が足りない → 工期延び、のジレンマ
- → **「同じ人数で今以上の成果」が経営の最重要課題に**

### 生産性ギャップ
- 建設業の労働生産性: **2,885円/時間**（全産業4,642円の62%）
- 年間労働時間: 建設業**1,966時間** vs 全産業1,651時間（+315時間）
- → 長時間働いているのに生産性が低い = **仕組みの問題**

### DX導入の実態
- DX導入率: **約2割**、6割以上が「予定なし」
- 導入企業も「使いこなせていない」が実態
- 建設業の**71.4%が従業員4人以下** — ITリテラシーの壁

### リフォーム市場
- 市場規模: **約7.3兆円**（2024年）、政府目標2030年**14兆円**
- 受注高は前年同期比**+5.9%**で堅調
- 仕事は増えるが人は減る = **AI活用の必然性が高まる**

### 【人材課題 × AI顧問の訴求ポイント】
| 人材課題 | AI顧問のソリューション |
|---------|---------------------|
| 採用難 | 既存社員の生産性向上 → 少人数で回せる体制 |
| 育成コスト | 営業教育AI → 教育時間の短縮 |
| 若手離職 | 書類自動化 → 残業削減 → 労働環境改善 |
| 技能伝承 | ナレッジBot → 属人知識のデジタル化 |
| 2024年問題 | 業務効率化 → 残業なしで同じアウトプット |

## 6. セッション振り返り（2026-02-18）

### 最大の気づき

> **選んだ3つのデモツール（AI見積もり・工程表・営業通信簿）は、
> 「バックオフィスの効率化」ではなく「営業・現場で直接活きる」「売上に直結する」ものだった。**
> これは今までなかったアイデアであり、非常に強い切り口。

### 営業通信簿AIの特別なポジション

> **自分のストーリーとAIが完全にかみ合うツール。**
>
> - トップセールスとしての実体験（5年間・年商2億超）
> - 前職リフォーム店の研修センターでの営業教育ノウハウ
> - 失注要因7分類・営業4ステップなどの独自フレームワーク
>
> → これらが**直接的にAIの教師データ・評価ロジック**になる。
> → 「AIを作れる人」はいても「営業の何を評価すべきか」を知っている人は少ない。
> → **ドメイン知識 × AI実装力 = 代替不可能なポジション**
