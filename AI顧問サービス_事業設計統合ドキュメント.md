### 4-2. 推奨ターゲット TOP3

| 選定理由 | 詳細 |
|---------|------|

| AI活用との相性が高い | 書類作成・情報収集・顧客対応＝AIの得意領域 |
| 支払能力が安定 | 顧問報酬モデルで収入安定。月額15万円は許容範囲 |
| 口コミ拡散力 | 士業同士のネットワークが濃密 |
| 危機感が強い | 「AIに仕事を奪われる」議論が活発 → 導入動機が強い |
| 市場規模 | 税理士約2.6万 + 社労士約2.7万 + 弁護士約1.8万 = 7万超 |

#### 第2位: 建設業（中小ゼネコン・サブコン）

- 90万〜128万人不足という圧倒的な人手不足の痛み
- 2024年問題（残業規制）の追い風
- 47万事業所というボリューム
- **リスク**: IT投資への抵抗感、現場重視の文化

#### 第3位: 中小製造業（従業員50〜300人）

- 品質検査・予知保全・生産計画でAI導入効果が明確
- 87%がパイロットPJ開始済み
- **リスク**: スマートファクトリー系専門ベンダーが間接競合

## 5. 顧客ニーズ・ジョブ理論

### 5-1. AI導入の失敗パターン TOP5

1. **「目的なき導入」症候群**: 話題だからで導入 → 何に使うか未定 → 定着しない
2. **「誰も使わない」問題**: ツール導入するも現場が使わず → 元に戻る
3. **「効果が見えない」迷走**: ROI設計なし → 成果不明 → 経営層の支持喪失
4. **「AI万能」幻想**: 過剰な期待 → ギャップに失望 → 「AIは使えない」結論
5. **「情報格差」の壁**: 大企業にはベンダーから情報が来るが中小には来ない

### 5-2. ジョブ理論分析

#### 機能的ジョブ（業務課題を解決したい）

| 優先度 | ジョブ | 具体例 |
|--------|------|--------|
| ★★★ | 定型業務を自動化したい | 議事録、メール、請求書、日報 |
| ★★★ | 情報収集・分析を効率化したい | 市場調査、法改正チェック、競合分析 |
| ★★ | 顧客対応を効率化したい | FAQ対応、チャットボット、営業資料 |
| ★★ | 人材不足を補いたい | 採用効率化、新人教育、マニュアル作成 |
| ★ | 新規事業を企画したい | AI活用の新商品開発 |

#### 感情的ジョブ（不安を解消したい）

| 優先度 | ジョブ | 心理状態 |
|--------|------|----------|
| ★★★ | AI時代に取り残される恐怖を解消したい | 「周りが進んでいるのに遅れている」焦り |
| ★★★ | 「何もわからない」不安を解消したい | 全体像が把握できず判断不能の閉塞感 |
| ★★ | 失敗したくない | 「高いツール入れて成果出なかったら」 |
| ★★ | 信頼できる相談相手が欲しい | ベンダーは売りつけ、社内に詳しい人ゼロ |

#### 社会的ジョブ

| 優先度 | ジョブ |
|--------|------|
| ★★ | DX先進企業として見られたい |
| ★★ | 業界内で「先を行っている」存在になりたい |

### 5-3. 核心インサイト

```
顧客が雇いたいのは「AI顧問」ではない。
「AIで確実に成果を出してくれる、業界を理解したガイド」である。

求めているもの:
1. 自分の業界・業務を理解している       ← 既存15社はここまで
2. 何をすべきか明確に教えてくれる       ← 既存15社はここまで
3. 実際にやってくれる（設定代行）       ← 圧倒的に不足
4. 成果が数字で見える                  ← 圧倒的に不足
5. 最終的には自分たちでできるようになる   ← 圧倒的に不足

→ 3〜5が勝機。
```

### 5-4. 購買トリガー TOP3

1. 同業事務所がAI導入して「繁忙期なのに定時で帰れるようになった」と聞いた
2. 電帳法・インボイス等の法改正対応に追われ「もう手作業では限界」と感じた
3. 顧問先から「AIで何かできないか」と相談され、答えられなかった

### 5-5. 比較対象（何と比較検討されるか）

```
1. 何もしない（最大の競合）    ← 60%以上がここ
2. 自社で独学（YouTube/Udemy）
3. SaaSツール直接導入
4. フリーランスAIコンサル
5. 他のAI顧問サービス
6. 大手コンサル（Accenture等）
7. 社内AI人材の採用
```

#### 本サービスのゲインクリエイター
1. AIツールの初期設定・カスタマイズまで代行
2. 士業の成功事例ライブラリ（同業実績で安心感）

## 7. サービス設計

### 7-1. メインプラン: 3ヶ月AI導入プログラム

| 項目 | 内容 |
|------|------|
| 月額 | 15万円（税抜） |
| 期間 | 3ヶ月（卒業型） |
| 総額 | 45万円（税抜） |
| MTG | 月2回（各60分・オンライン） |
| チャット | 無制限（Slack/ChatWork） |
| 実装支援 | AIツール設定・カスタマイズ代行（月4時間） |
| 成果物 | AI活用マニュアル、業務フロー改善書、効果測定レポート |

### 7-2. 3ヶ月プログラム構成

```
Month 1: 診断・設計
  Week 1: 初回ヒアリング + 現状業務棚卸し
  Week 2: AI導入計画策定 + ツール選定
  Week 3: 第1業務へのAI導入（ハンズオン）
  Week 4: 定着確認 + 第2業務の準備

Month 2: 実装・展開
  Week 5-6: 第2・第3業務へのAI導入
  Week 7-8: 全スタッフへの展開 + 社内研修

Month 3: 定着・卒業
  Week 9-10: 自走テスト（サポートを徐々に減らす）
  Week 11: 効果測定レポート作成
  Week 12: 卒業MTG + 今後のロードマップ提示
```

## 8. 収益モデル・ユニットエコノミクス

### 8-1. 価格設定の根拠（3方向からの検証）

| アプローチ | 算出結果 |
|-----------|---------|
| コストベース（原価×粗利率40%） | 月額20万円 |
| 競合ベース（ボリュームゾーン+特化プレミアム50%） | 月額15万円 |
| バリューベース（顧客価値10万円/月×1.5倍） | 月額15万円 |

→ **月額15万円**を採用

### 8-2. ユニットエコノミクス

```
【初期3ヶ月（プログラム期間）】
売上:   15万円 × 3ヶ月 = 45万円
変動費: 8h/月 × 3ヶ月 × 1.5万円/h = 36万円
粗利:   9万円（粗利率20%）

【卒業後サポート（継続した場合）】
売上:   5万円/月 × 平均6ヶ月 = 30万円
変動費: 2h/月 × 6ヶ月 × 1.5万円/h = 18万円
粗利:   12万円（粗利率40%）

【LTV】= 粗利9万 + 粗利12万 = 21万円
【CAC】= 5万円（加重平均）
【LTV/CAC比率】= 4.2倍 ✅（健全ライン3倍以上）
```

## 11. リスクと対策

### 11-1. クリティカルサクセスファクター（CSF）

| # | CSF | 理由 |
|---|-----|------|
| 1 | MVP顧客で実証データを作る | 「30%削減」の実績がなければ全てが空論 |
| 2 | 士業業務の深い理解 | 浅い知識では全業種対応の既存競合と同じ |
| 3 | 実装まで含む支援 | 「相談だけ」では差別化が効かない |
| 4 | 卒業後の継続率70%以上 | LTV/CACの健全性がここにかかっている |
| 5 | 紹介による顧客獲得 | 士業ネットワークを活かせればCAC大幅削減 |

### 11-2. 主要リスクと対策

| リスク | 影響度 | 対策 |
|--------|--------|------|
| MVP参加者が集まらない | 高 | 知人ネットワーク活用+無料提供で障壁を下げる |
| 3ヶ月で成果が出ない | 高 | 対象業務を「議事録作成」1つに絞り確実に70%削減を見せる |
| 卒業後の解約率が高い | 中 | コミュニティで繋がりを維持 |
| 競合が士業特化で参入 | 中 | 早期に事例・ノウハウ・コミュニティを蓄積しスイッチングコスト構築 |
| 士業のAI抵抗感 | 低 | 「代替」ではなく「付加価値向上」メッセージ |
| AI技術の急速な変化 | 低 | ツール非依存の「活用メソッド」を核にする |
| 損益分岐への到達遅延 | 中 | テンプレート化で変動費削減、グループプログラムで効率化 |
