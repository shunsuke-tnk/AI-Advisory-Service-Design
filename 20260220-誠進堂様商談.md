### ページ5: 建設業の現場課題① — 業務効率（共感ページ）

**設計意図**: 「この人は現場をわかっている」と自然に感じてもらう。業務の非効率を具体的に描写し、共感を得る。

**書類・事務**
- 現場から帰社して、そこから見積もり作成。気づけば深夜になっている
- 現場で紙に記録したことを、事務所でPCに打ち直す二重作業
- FAXで届いた見積もりを、一つずつ手で転記する作業
- 小さな申請も全て紙。デジタル化が進んでいない

**営業・見積もり**
- 見積もり1件に2〜3時間。原価調査、職人への連絡、FAX待ち、転記、積算と工程が多い
- 効率を優先すると提案品質が下がる。時短と品質の両立が難しい
- 新人は「何を聞けばいいかすらわからない」。教育に膨大な時間がかかる

**現場管理・コミュニケーション**
- 下請業者との電話連絡が多すぎる
- お客様×業者×自分の三者日程調整
- 現場に行かないと状況がわからない

### ページ6: 建設業の現場課題② — 人材・組織（共感ページ）

**設計意図**: 誠進堂の最大課題である「人材」にフォーカス。成長フェーズ特有の痛みに深く刺さるページ。

**人材の採用・定着・教育**（★誠進堂の最大課題）
- 有効求人倍率4.81倍。採りたくても人が採れない
- 新卒3年離職率は大卒30%、高卒40〜50%。若手が定着しない
- 営業ノウハウが属人化し、トップセールスの技が組織に伝承されない
- 教育に時間をかけたくても、教える側も現場で手一杯

**ツール導入の壁**
- 新しいツールを導入しても、現場スタッフがデジタルに慣れておらず定着しない
- 「入れたけど結局使われていない」ツールが増えていく
- 就業者の36.6%が55歳以上。ITリテラシーの壁は想像以上に高い

> *締め: 「これらは私が5年間、毎日向き合ってきた課題です。そして、AIで解決できる課題でもあります。」*

### ページ7: 「効率化のAI」と「売上を上げるAI」は、まったく別物です

**設計意図**: 汎用コンサルを否定せず、「売上を上げるAIは業界特化でなければ作れない」という構造を示し、特化型に任せたいと自然に思わせる。

**核心メッセージ**: AI活用には2種類ある

| | 効率化のAI | 売上を上げるAI |
|--|-----------|--------------|
| やること | 作業を速くする | 契約率・単価・粗利を上げる |
| 必要な知識 | 業務フローの理解 | **業界の商習慣＋営業ロジック** |
| 誰が作れるか | 汎用のAI会社でもできる | **業界で売ってきた人間にしかできない** |

**なぜ業界特化でなければできないのか — 2つの理由**

1. **業界の痛みを知っている** → だから「何を解決すべきか」がわかる
   - 建設業の現場で何が非効率なのか、どこでつまずくのか、なぜAI導入が進まないのか
   - 5年間、毎日現場で向き合ってきたからこそ見える課題がある

2. **売り方を知っている** → だから「勝てる営業ロジック」をAIに入れられる
   - 5年連続 年間売上1億8,000万円、粗利率40%を維持した営業手法
   - このロジックをAIに組み込むことで、効率化だけでなく売上・利益を上げるツールが作れる

### ページ8: 見積もり作成AI

**設計意図**: 機能の魅力を「現場の体験」として伝える。スペック羅列ではなく、使うとどう変わるかを直感させる。

**キャッチ**: 見積もり作成、10分で完成。現場で終わる。

**主要機能**
- **音声入力対応** — 現場で話すだけで見積もりの下書きが完成。帰社後の作業がなくなる
- **PDF/FAX読取** — 届いた見積書をそのまま読み込み。手作業の転記が不要に
- **3パターン自動生成** — 松竹梅の提案を自動作成。お客様に選んでもらう営業ができる
- **原価自動積算** — 過去データと市場単価から適正原価を算出。粗利の取りこぼしを防ぐ

**現場での使い方イメージ**
```
① 現場でお客様と話しながら音声入力
② AIが即座に3パターンの見積書を生成
③ その場でお客様に提示 → 「持ち帰って検討」がなくなる
④ 帰社後の残業ゼロ
```

> *ポイント: 「2〜3時間かかっていた見積もりが10分」は、デモで実際に見せる*

### ページ9: 営業通信簿

**設計意図**: 「AIが営業を採点する」ではなく、「失注の構造を可視化し、次のアクションが明確になる」ツールとして見せる。松村社長・松田取締役はトップセールス出身 → 「テストクロージング」の一言で共感を得る。

**キャッチ**: 失注には、7つの原因がある。1つでも欠けたら、契約は取れない。

**評価フレームワーク: ToC営業の7つの失注要因**

> 商談前にテストクロージングで7つの要因を潰す。これがトップセールスの基本動作。
> このAIは「テストクロージングできていたか」を要因ごとに判定する。

| 順 | 要因 | チェック観点 |
|----|------|-------------|
| ① | **時期** | 「今すぐ」になっているか |
| ② | **自分** | 「この人から買いたい」と思われているか |
| ③ | **キーマン** | 決裁者が揃っているか |
| ④ | **金額** | 金額に対する合意が取れているか |
| ⑤ | **プラン** | 提案内容に納得しているか |
| ⑥ | **競合** | 競合状況を把握し対策できているか |
| ⑦ | **自社** | 自社への信頼があるか |
| +1 | **先回りクロージング** | 契約意思の事前合意を取れているか |

**AIの仕組み（技術的な差別化ポイント）**
- 7つの要因ごとに**専門AIが個別に判定**（1AI＝1要因で並列稼働）
- 長文を総合評価するのではなく、「テストクロージングできたか否か」をシンプルに判定
- → 尖った精度の評価が可能。汎用AIの「なんとなく評価」とはまったく異なる

**出力イメージ**
- **弱み分析**: 7要因のどこが未達かを可視化
- **即決オファー提案**: 商談の場で使えるパターンをAIが提示
- **反論パターン**: お客様の断り文句に対する切り返し
- **御社独自ロジック**: 誠進堂の商材・顧客層に合わせたカスタマイズ

> *ポイント: 具体的なテクニック（各要因の埋め方・クロージングスクリプト）は導入後にカスタマイズして組み込む。資料上は「構造」の提示にとどめる。*

### ページ10: こんなこともできます（事例展開）

**設計意図**: ページ8・9のフック2本に加え、「他にもこれだけ作れる」と幅を見せる。3本ともヒアリングで決める前提なので、選択肢の豊富さを示すことで「自分たちに合うものを選べる」と感じさせる。

**事例リスト**（箇条書きで簡潔に）

- **二重転記の自動化** — 報告書・小口精算・日報・顧客管理シート。フォームや音声で入力するだけで所定書類に自動転記
- **カタログのデジタル化** — 全カタログを独自Webページとして一括整理。軽量で検索可能、現場でもすぐ参照できる

- **提案資料の自動生成** — 少ない入力で完璧な見た目の提案資料が完成。新人でもプロ品質の資料を出せる
- **遠隔現場管理** — 音声日報・工程表の自動生成・リアルタイム更新。お客様にも共有でき、「今何をやっているか」が見える化

> *ポイント: 詰め込みすぎず、1項目1〜2行で「こんなこともできるのか」という発見を与える。深掘りは商談の会話の中で行う。*

### ページ13: 投資対効果② — なぜこの数字が出せるのか

**設計意図**: ROI試算の直後に「根拠」を示す。田中の実績が唯一無二のエビデンスになる。

> 田中の実績:
> - リフォーム営業として**粗利率40%を維持**し続けた
> - **年間売上1億8,000万円を5年連続で達成**
> - 見積もり作成時間: 2〜3時間 → 10分に短縮
>
> **このロジック（高粗利×高契約率×効率化）を、今回のAIツールで再現しています。**
>
> つまり、「AIで何ができるか」の提案ではなく、
> 「実際に結果を出してきた営業手法を、AIで御社全体に展開する」提案です。
